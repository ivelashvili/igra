# База данных - Единый источник истины

## Обзор

Игра теперь использует SQLite базу данных как единый источник истины для всех данных. Это гарантирует, что:
- Все клиенты (веб-интерфейс и Mini App) видят одинаковые данные
- Состояние игры сохраняется между перезапусками сервера
- Нет рассинхронизации между разными запросами

## Структура базы данных

### Таблицы

1. **games** - Информация об играх
2. **players** - Игроки
3. **player_resources** - Ресурсы игроков
4. **buildings** - Объекты игроков
5. **current_prices** - Текущие цены на ресурсы
6. **resource_prices** - История цен (по раундам)
7. **round_events** - События раундов
8. **player_actions** - Действия игроков (для расчета спроса/предложения)
9. **game_version** - Версия игры (для оптимистичных блокировок)

## Как это работает

### Автоматическое сохранение

Все изменения состояния игры автоматически сохраняются в БД:
- При добавлении игрока
- При покупке/продаже ресурсов
- При строительстве объектов
- При продаже объектов
- При обработке раунда

### Автоматическая загрузка

При каждом запросе к API:
1. Проверяется, загружена ли игра в память
2. Если нет - загружается из БД
3. Если да - обновляется из БД (для синхронизации)

### Синхронизация

- **Веб-интерфейс**: Обновляется через WebSocket каждую секунду, данные читаются из БД
- **Mini App**: Обновляется через polling каждые 2 секунды, данные читаются из БД

## Использование

### Инициализация БД

```python
import database
database.init_database()  # Создает все таблицы
```

### Создание игры

```python
from game_engine import Game

# Создается автоматически в БД
game = Game(num_players=30)
```

### Загрузка игры из БД

```python
# При старте сервера игра загружается автоматически
# Или вручную:
game = Game(num_players=30, game_id=game_id, load_from_db=True)
```

### Сохранение состояния

```python
# Автоматически при каждом изменении
# Или вручную:
game.save_to_database()
```

### Обновление из БД

```python
# Обновляет состояние из БД
game.load_from_database()
```

## Преимущества

1. **Единый источник истины** - все данные в одном месте
2. **Персистентность** - данные сохраняются между перезапусками
3. **Синхронизация** - все клиенты видят актуальные данные
4. **Атомарность** - транзакции предотвращают race conditions
5. **Масштабируемость** - можно добавить несколько игр

## Файл базы данных

База данных хранится в файле `royal_exchange.db` в корне проекта.

## Миграции

При изменении структуры БД нужно:
1. Обновить `database.init_database()`
2. Удалить старый файл `royal_exchange.db` (или создать миграцию)
