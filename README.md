# Королевская Биржа

Игра для корпоративных мероприятий, где участники торгуют ресурсами, строят объекты и соревнуются за максимальную капитализацию.

## 🎮 Описание

**Королевская Биржа** — экономическая стратегия для 5–30 игроков, 10 раундов. Игроки покупают ресурсы, строят объекты, получают доход и соревнуются за звание самого успешного торговца.

### Особенности

- 📊 **Веб-интерфейс для проектора** — отображение состояния игры в реальном времени
- 📱 **Telegram Mini App** — мобильный интерфейс для игроков
- 🎬 **Видео-введение** — погружение в атмосферу игры
- 💾 **Персистентность** — данные хранятся в PostgreSQL или SQLite
- 🔄 **Система снимков** — откат на предыдущие раунды
- ⚡ **Синхронизация в реальном времени** — WebSocket для проектора
- 🛡️ **Админ-панель** — создание игр, управление игроками и конфигом

## 🚀 Быстрый старт

### Требования

- Python 3.9+
- PostgreSQL (рекомендуется для продакшена) или SQLite (по умолчанию)

### Установка

1. Клонируйте репозиторий или используйте эту папку.

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. (Опционально) Настройте базу данных:
   - **SQLite** — по умолчанию, файл `royal_exchange.db` создаётся автоматически.
   - **PostgreSQL** — создайте базу и задайте переменную окружения:
     ```bash
     export DATABASE_URL="postgresql://user:password@host:5432/dbname"
     ```

4. Запустите сервер — база инициализируется при старте:
```bash
python3 run_web.py
```

Сервер будет доступен на `http://localhost:8000`.

### Доступ

| Интерфейс | URL |
|-----------|-----|
| **Проектор (веб)** | `http://localhost:8000` |
| **Telegram Mini App** | `http://localhost:8000/miniapp` |
| **Тестовый Mini App** | `http://localhost:8000/miniapp_test?player=1` (игрок без Telegram) |
| **Админ-панель** | `http://localhost:8000/admin` |

Игра привязана к **коду игры** (6 цифр). Код вводится на проекторе и в Mini App; создаётся в админке.

## 🎯 Основные компоненты

### Игровой движок (`game_engine.py`)

- Раунды и фазы (события → доходы → закупки)
- Действия игроков: покупка/продажа ресурсов, строительство/продажа зданий
- Расчёт доходов от объектов с учётом событий и насыщения рынка
- Сохранение и загрузка состояния из БД

### База данных (`database.py`)

- Поддержка **PostgreSQL** (asyncpg) и **SQLite** (aiosqlite)
- Таблицы: игры (с кодом 6 цифр), игроки, ресурсы, здания, снимки, конфиг
- Игры загружаются по запросу по `game_code`; состояние сохраняется после действий и смены раунда

### Веб-сервер (`web_server.py`)

- REST API для проектора и Mini App (все эндпоинты принимают `game_code`)
- WebSocket `/ws?game_code=...` для обновлений в реальном времени
- Раздача статики и шаблонов
- Глобальная обработка ошибок и валидация (Pydantic, Telegram initData)

## 🎮 Игровой процесс

### Раунды

1. **Фаза событий** — применяются события (фиксированный сценарий), влияющие на цены и доходы
2. **Фаза доходов** — активные объекты приносят ресурсы/монеты
3. **Фаза закупок** — игроки покупают/продают ресурсы и строят объекты
4. **Переход** — обновление цен по спросу/предложению, следующий раунд

### Ресурсы

Камень, Дерево, Железо, Скот, Овощи, Рабы, Золото, Зерно, Рыба.

### Объекты (11 типов)

Лесоповал, Каменоломня, Рыболовня, Теплицы, Ферма, Посевные поля, Кузнечная, Постоялый двор, Трактир, Куртизанские палатки, Золотой рудник. Каждый приносит доход в ресурсах или монетах.

## 🔧 API (основное)

Все запросы, где применимо, принимают query-параметр `game_code` (6 цифр).

### Проектор

- `GET /` — главная страница
- `GET /api/leaderboard?game_code=...` — турнирная таблица
- `GET /api/prices?game_code=...` — цены на ресурсы
- `GET /api/buildings?game_code=...` — статистика по объектам
- `GET /api/round/{round_number}/summary?game_code=...` — сводка раунда
- `POST /api/game/next-round?game_code=...` — следующий раунд
- `GET /api/game/snapshots?game_code=...` — список снимков
- `POST /api/game/rollback?game_code=...` — откат на раунд
- `WebSocket /ws?game_code=...` — обновления в реальном времени

### Mini App

- `GET /api/miniapp/player/state?game_code=...` — состояние игрока
- `GET /api/miniapp/prices?game_code=...` — цены
- `GET /api/miniapp/leaderboard?game_code=...` — рейтинг
- `POST /api/miniapp/player/buy-resource`, `sell-resource`, `build`, `sell-building` — действия игрока (с авторизацией Telegram)

### Админка и мониторинг

- `GET /admin` — админ-панель (логин по токену)
- `GET /health`, `/health/ready`, `/metrics` — здоровье и метрики

## 🧪 Ручное тестирование

1. Запустите сервер: `python3 run_web.py`
2. Откройте админку: `http://localhost:8000/admin` — создайте игру и получите код.
3. Откройте проектор с этим кодом и тестовый Mini App в нескольких вкладках:
   - `http://localhost:8000/miniapp_test?player=1`
   - `http://localhost:8000/miniapp_test?player=2`
   - …
4. В тестовом интерфейсе можно использовать кнопку «Создать всех игроков».

## 🚢 Деплой (Railway и др.)

1. Подключите репозиторий к [Railway.app](https://railway.app) (или аналогу).
2. Укажите переменные окружения: `DATABASE_URL` (для PostgreSQL), при необходимости `PORT`.
3. Railway по умолчанию использует `Procfile`: `web: uvicorn web_server:app --host 0.0.0.0 --port $PORT`
4. Настройте Telegram Mini App на URL вашего приложения.

## 🔐 Безопасность

- Валидация входных данных (Pydantic)
- Проверка авторизации через Telegram WebApp initData
- Админ-панель защищена токеном

## 📊 Технологии

- **Backend:** Python 3.9+, FastAPI, WebSocket
- **БД:** PostgreSQL (asyncpg) или SQLite (aiosqlite)
- **Frontend:** HTML, CSS, JavaScript
- **Деплой:** Railway (или любой хостинг с поддержкой Procfile/uvicorn)

## 🤝 Поддержка

При проблемах:

1. Проверьте логи сервера (в консоли и при необходимости в `logs/`).
2. Убедитесь, что `DATABASE_URL` задана корректно (для PostgreSQL) или что доступна запись в текущую директорию (для SQLite).
3. Для админки и Mini App проверьте, что используете правильный `game_code` (6 цифр).
