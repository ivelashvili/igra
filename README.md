# Королевская Биржа

Игра для корпоративных мероприятий, где участники торгуют ресурсами, строят объекты и соревнуются за максимальную капитализацию.

## 🎮 Описание

**Королевская Биржа** - это экономическая стратегия для 5-30 игроков, состоящая из 10 раундов. Игроки покупают ресурсы, строят объекты, получают доход и соревнуются за звание самого успешного торговца.

### Особенности

- 📊 **Веб-интерфейс для проектора** - отображение текущего состояния игры в реальном времени
- 📱 **Telegram Mini App** - мобильный интерфейс для игроков
- 🎬 **Видео-введение** - погружение в атмосферу игры
- 💾 **Персистентность** - все данные сохраняются в базе данных
- 🔄 **Система снимков** - возможность отката на предыдущие раунды
- ⚡ **Синхронизация в реальном времени** - WebSocket для веб-интерфейса

## 🚀 Быстрый старт

### Установка

1. Клонируйте репозиторий или используйте эту папку

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Инициализируйте базу данных:
```bash
python3 -c "import database; database.init_database()"
```

### Запуск

```bash
python3 run_web.py
```

Сервер запустится на `http://localhost:8000`

### Доступ

- **Веб-интерфейс (проектор)**: `http://localhost:8000`
- **Telegram Mini App**: `http://localhost:8000/miniapp`
- **Тестовый Mini App**: `http://localhost:8000/miniapp_test?player=1`

## 🎯 Основные компоненты

### Игровой движок (`game_engine.py`)

Управляет всей логикой игры:
- Раунды и фазы игры
- Действия игроков (покупка/продажа ресурсов, строительство)
- Расчет доходов от объектов
- Обработка событий

### База данных (`database.py`)

SQLite база данных как единый источник истины:
- Автоматическое сохранение всех изменений
- Синхронизация между веб-интерфейсом и Mini App
- Персистентность между перезапусками

### Веб-сервер (`web_server.py`)

FastAPI приложение с:
- REST API для веб-интерфейса и Mini App
- WebSocket для обновлений в реальном времени
- Статические файлы (HTML, CSS, JS, изображения, видео)

## 🎮 Игровой процесс

### Раунды

1. **Фаза событий** - применяются события, влияющие на цены и доходы
2. **Фаза начисления доходов** - активные объекты приносят доход
3. **Фаза закупок** - игроки покупают/продают ресурсы и строят объекты
4. **Обновление состояния** - переход к следующему раунду

### Ресурсы

- Камень, Дерево, Железо, Скот, Овощи, Рабы, Золото, Зерно, Рыба

### Объекты

11 типов объектов, каждый приносит доход в ресурсах или монетах:
- Лесоповал, Каменоломня, Рыболовня, Теплицы, Ферма, Посевные поля
- Кузнечная, Постоялый двор, Трактир, Куртизанские палатки, Золотой рудник

## 🔧 API Endpoints

### Веб-интерфейс

- `GET /` - Главная страница
- `GET /api/leaderboard` - Турнирная таблица
- `GET /api/prices` - Цены на ресурсы
- `GET /api/buildings` - Статистика по объектам
- `POST /api/game/next-round` - Следующий раунд
- `GET /api/game/snapshots` - Список снимков
- `POST /api/game/rollback` - Откат на раунд
- `WebSocket /ws` - Обновления в реальном времени

### Mini App

- `GET /api/miniapp/player/state` - Состояние игрока
- `GET /api/miniapp/prices` - Цены на ресурсы
- `GET /api/miniapp/leaderboard` - Рейтинг
- `POST /api/miniapp/player/buy-resource` - Купить ресурс
- `POST /api/miniapp/player/sell-resource` - Продать ресурс
- `POST /api/miniapp/player/build` - Построить объект
- `POST /api/miniapp/player/sell-building` - Продать объект
